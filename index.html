This is for Git practice. Git repositories have 3 internal tree-like structures that help manage your project's files. Also, get me outta here, please!

  • Working Directory: This is where you do your work. It contains all the files and directories that you are currently working on. Changes you make here are not yet saved in Git.
  • Staging Area: This is where you prepare your changes to be committed. You can add changes to the staging area using the git add command, which lets you select which changes to include in the next commit.
  • Commit Log: This is a history of all the commits you've made in your repository. Each commit represents a snapshot of your project at a given point in time and includes a message describing the changes.
